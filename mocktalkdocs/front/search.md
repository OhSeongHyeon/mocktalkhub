# 검색 UI/UX

## 범위
- 통합검색 페이지: `/search`
- 상단 글로벌 검색창: `TopMenuBar`
- 게시판 상세 내부 게시글 검색: `BoardArticlePanel`

## 라우트 / 쿼리 규약
- `q`: 검색어
- `type`: `ALL | BOARD | ARTICLE | COMMENT | USER`
  - 기본값: `ALL`
- `order`: `LATEST | OLDEST`
  - 기본값: `LATEST`
- `page`: 0부터 시작
  - 기본값: `0`
- `size`: 페이지 크기
  - 기본값: `10`
  - 선택값: `10 / 20 / 30 / 40 / 50`
- `boardSlug`: 게시판 상세 내부 검색일 때만 사용

## 상단 검색창 동작
- `TopMenuBar`에서 검색어를 입력하고 실행하면 `/search`로 이동한다.
- 검색어가 있으면 기본 쿼리는 아래와 같다.
  - `/search?q=...&type=ALL&order=LATEST&page=0`
- 검색어가 비어 있으면 단순히 `/search`로 이동한다.

## 검색 페이지 구조
- `AppShell + PageContainer(width="auto")`를 사용한다.
- 페이지 상단은 `PageHeader`로 시작한다.
  - eyebrow: `검색`
  - title: `통합검색`
  - description: `게시판, 게시글, 댓글, 사용자를 한 번에 찾을 수 있습니다.`
- `PageHeader` 내부 보조 블록에 아래 입력을 배치한다.
  - 검색어 입력
  - 검색 범위 칩
  - 정렬 칩
  - 표시 개수 선택
- 에러는 `ui-state ui-state-danger`로 헤더 아래에 노출한다.
- 로딩 중에는 `검색 결과를 불러오는 중입니다...` 문구를 보여준다.

## 상태 동기화 규칙
- `SearchPage.vue`는 현재 라우트 쿼리에서 상태를 복원한다.
- 라우트 쿼리가 바뀌면 다시 검색한다.
- 검색 범위, 정렬, 표시 개수를 바꾸면 `page=0`으로 리셋한다.
- 공백만 있는 검색어는 실행하지 않는다.

## 결과 표시 방식

### `type=ALL`
- 게시판, 게시글, 댓글, 사용자 4개 섹션을 한 화면에 요약 노출한다.
- 각 섹션은 `SectionHeader`를 사용한다.
- 섹션별 `hasNext`가 있을 때만 `더보기` 버튼을 노출한다.
- `더보기`를 누르면 해당 타입 단일 결과 화면으로 전환한다.
- 섹션별 결과가 없으면 `‘검색어’ 검색 결과가 없습니다.` 문구를 보여준다.

### `type=BOARD | ARTICLE | COMMENT | USER`
- 선택한 타입 결과만 보여준다.
- 상단에 `SectionHeader`로 현재 타입 결과임을 표시한다.
- 결과가 없으면 `ui-state ui-state-empty`로 `‘검색어’ 검색 결과가 없습니다.`를 보여준다.
- 상단 보조 정보는 아래만 노출한다.
  - 현재 페이지 번호
  - `hasNext=true`일 때 `다음 페이지 있음`
- 전체 건수나 전체 페이지 수는 표시하지 않는다.

## 타입별 카드 정보
- 게시판
  - 게시판 이미지
  - 게시판명
  - 공개 범위 라벨
  - 설명
- 게시글
  - 게시판명
  - 작성자
  - 작성일
  - 제목
- 댓글
  - 게시판명
  - 원문 게시글 제목
  - 작성자
  - 댓글 내용
- 사용자
  - `@handle`
  - 표시 이름

## 페이지네이션
- 타입 단일 결과 화면에서만 페이지네이션을 노출한다.
- 페이지 번호는 10개 단위 윈도우를 사용한다.
- 버튼 구성은 아래와 같다.
  - `이전`
  - `처음`
  - `이전 10`
  - 현재 윈도우 페이지 번호
  - trailing `…`
  - `다음 10`
  - `다음`
- 마지막 페이지는 알 수 없으므로 `hasNext` 기준으로만 다음 이동과 `…`를 제어한다.

## 게시판 상세 내부 검색
- 게시판 상세의 `BoardArticlePanel`도 같은 검색 API 래퍼를 사용한다.
- 호출 파라미터는 아래와 같다.
  - `type=ARTICLE`
  - 현재 정렬값
  - 현재 페이지
  - 현재 표시 개수
  - 현재 `boardSlug`
- 검색 모드에서는 공지 영역을 비우고 검색 결과만 목록에 노출한다.
- 이 경우에도 `hasNext`, `hasPrevious`만 사용하고 전체 페이지 수는 계산하지 않는다.

## 프론트 검색어 처리 원칙
- 프론트는 검색어를 `trim()` 한 뒤 사용한다.
- 공백만 있는 입력은 요청하지 않는다.
- 토큰 분리, 연산자 해석 같은 검색 해석 로직은 프론트에서 수행하지 않는다.

## 문구 기준
- 로딩: `검색 결과를 불러오는 중입니다...`
- 실패: API 메시지를 우선 사용하고, 없으면 `검색에 실패했습니다.`
- 빈 상태: `‘검색어’ 검색 결과가 없습니다.`
